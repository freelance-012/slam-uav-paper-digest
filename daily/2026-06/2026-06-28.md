## 🟢 Tier-1: CPU 实时就能跑（8 篇）

### 1. [SA-LIVO: Efficient LiDAR-Inertial-Visual Odometry with Subspace-Aware Degeneracy Handling](https://arxiv.org/abs/2606.25699)
- **方向**: 📡 多传感器融合, 🎯 紧耦合后端
- **门槛**: CPU 实时（笔记本 CPU 12.3ms/帧，嵌入式 ARM 26.8ms/帧，无 GPU）
- **核心 idea**: 对 LiDAR-Visual 联合信息矩阵做特征分解，逐特征方向软门控，退化方向衰减、可观方向全保留
- **关键点**: InEKF 共享线性化点，photometric Jacobian 只组装一次循环复用；29 序列三基准测试；代码开源
- **对工作的价值**: ⭐⭐⭐⭐⭐
- **摘要**: 紧耦合 LIVO 融合了 LiDAR 精准深度和视觉互补信息，但两种外感受器各自退化模式不同——LiDAR 在几何欠约束时退化，视觉在光照/纹理缺失时退化。现有方案在模态层面做退化检测和协方差膨胀，但忽略了信息矩阵的方向结构。SA-LIVO 提出子空间感知信息融合框架（SAIF），对联合信息矩阵特征分解后逐方向线性钳位软门控，在退化方向衰减的同时保留可观方向全强度。实验在 HILTI'22、New College、Oxford Spires 三个基准 29 条序列上验证，并在并发退化场景下保持有界漂移。嵌入式 ARM 板无 GPU 跑 26.8ms/帧，峰值内存降低 3.6-6.3 倍。

### 2. [FAR-LIO: Enabling High-Speed Autonomy through Fast, Accurate, and Robust LiDAR-Inertial Odometry](https://arxiv.org/abs/2606.26010)
- **方向**: 📡 多传感器融合, ⚙️ 轻量化
- **门槛**: CUDA 加速但算法本身是 EKF 后端，Jetson 级别可跑（IROS 2026）
- **核心 idea**: CUDA 加速 voxel hashmap + 稀疏感知 GICP + 自适应阈值，面向高速场景（250km/h 自动驾驶赛车验证）
- **关键点**: 四种传感器配置验证；位置误差降低 6.9%，运行时降低 38.4%；代码已开源 (github.com/TUMFTM/FAR-LIO)
- **对工作的价值**: ⭐⭐⭐⭐
- **摘要**: 高速自主场景中里程计延迟是闭环控制的关键瓶颈。FAR-LIO 提出基于 CUDA 加速的 LiDAR-惯性里程计框架，使用 CUDA voxel hashmap 实现并行最近邻搜索和高效地图更新，配合稀疏感知 GICP 和自适应阈值实现低延迟。EKF 后端通过上采样和延迟补偿融合 LiDAR 里程计与高频 IMU。在公开数据集和两辆自动驾驶赛车（最高 250km/h）上验证，单参数集位置误差降低 6.9%，运行时降低 38.4%。代码已开源。

### 3. [GeoFlow-SLAM++: A Robust Multi-Camera Visual-Inertial SLAM System with Relocalization](https://arxiv.org/abs/22051)
- **方向**: 📡 多传感器融合, 🔁 回环 / 位置识别
- **门槛**: CPU 实时（ORB 前端）；NN-Feature 前端需轻量 GPU
- **核心 idea**: 多相机 VI-SLAM，统一 body-centric 公式，支持 ORB / SuperPoint+LightGlue 双前端，跨视图重定位
- **关键点**: EuRoC/OpenLORIS/TUM/Hilti/自采多相机数据集验证；NN-Feature 前端在外观挑战场景鲁棒性显著提升；跨视图重定位达 LiDAR 级精度
- **对工作的价值**: ⭐⭐⭐⭐
- **摘要**: 单目和 RGB-D VI-SLAM 受限于视场角和传感器特定退化模式，跨 session 重定位不可靠。GeoFlow-SLAM++ 将 GeoFlow-SLAM 从单 RGB-D 扩展到标定多相机 rig，统一 body-centric 公式。支持 ORB 前端和 NN-Feature 前端（SuperPoint + LightGlue），将跟踪、建图、重定位统一在共享 body state 上。多相机重投影约束 + IMU 预积分 + 跨视图位置识别 + 双流光流/NN-Feature 跟踪。在 EuRoC、OpenLORIS、TUM、Hilti 和自采手持多相机数据集上评估，NN-Feature 前端提升外观挑战场景鲁棒性，多相机公式在 Hilti 上竞争力强，跨视图重定位在手持数据集上达 LiDAR 级性能。

### 4. [Ultra-Fusion: A Resilient Tightly-Coupled Multi-Sensor Fusion SLAM Framework](https://arxiv.org/abs/2606.21223)
- **方向**: 📡 多传感器融合, 🎯 紧耦合后端
- **门槛**: CPU 可跑（滑窗优化器，因子图级别）；含 GNSS/wheel 可选因子
- **核心 idea**: 统一滑窗估计器，时间戳排序异步测量转可选因子，支持 WIO/VIO/LIO/LVIO + wheel + GNSS
- **关键点**: 可观性感知初始化选择引导模式；因子级可靠性调度门控退化测量；在线 LiDAR-IMU 时空标定；60+ 开源系统对比，5 大基准；将开源
- **对工作的价值**: ⭐⭐⭐⭐⭐
- **摘要**: 智能交通系统中可靠定位至关重要，但紧耦合多传感器融合在传感器退化（光照差、LiDAR 退化、轮速打滑、GNSS 失效）和时空标定误差面前仍然脆弱。Ultra-Fusion 提出基于统一滑窗估计器的紧耦合多传感器定位框架，异步测量按时间戳排序转化为可选因子，支持 WIO/VIO/LIO/LVIO 并可选 wheel 和 GNSS 增强。可观性感知初始化选择引导模式，因子级可靠性调度门控退化测量，在线 LiDAR-IMU 时空标定在运行中精修时间偏移和旋转外参。在 M3DGR、M2DGR-Plus、KAIST、GrandTour、MARS-LVIG 五大基准上评估 60+ 开源系统，覆盖轮式、足式、空中平台。

### 5. [Invariant Kalman filtering for extended pose estimation in multi-IMU articulated rigid-body systems](https://arxiv.org/abs/2606.25083)
- **方向**: 📡 多传感器融合, 🎯 紧耦合后端
- **门槛**: CPU 实时（IEKF 滤波，纯矩阵运算）
- **核心 idea**: 相对 L-extended pose 李群表示，多 IMU 关节系统群仿射动力学，关节约束作为无噪伪测量嵌入 IterIEKF
- **关键点**: UR5e 机器人和人腿验证；RMSE 降低 50%+；收敛快、run-to-run 变异小
- **对工作的价值**: ⭐⭐⭐⭐
- **摘要**: 多 IMU 关节刚体系统的扩展位姿估计是机器人和人体运动分析的关键挑战。IEKF 对单刚体有收敛保证和一致性，但扩展到关节系统不平凡——体间位姿耦合阻碍直接应用，关节运动学约束在不变框架内表达仍是开放问题。本文提出相对 L-extended pose 李群表示，每个 body 一个 IMU，得到群仿射动力学并将关节约束表达为不变形式，作为无噪伪测量嵌入 IterIEKF。在 UR5e 机器人和人腿上验证，RMSE 比次优滤波降低至少 50%。

### 6. [Delta-Position Estimation-Based IMU Odometry: A Comparison of MLP and Kolmogorov-Arnold Networks](https://arxiv.org/abs/2606.25454)
- **方向**: 🧠 SLAM + 深度学习, ⚙️ 轻量化
- **门槛**: CPU 实时（KAN 仅 8444 参数，MLP 57859 参数）
- **核心 idea**: 用 KAN（B-spline 可学习激活）替代 MLP 做 IMU 增量位移估计，参数少 6.9 倍，漂移低 44%
- **关键点**: EuRoC MAV 数据集；50ms 滑窗 Δp 估计 + 数值积分；KAN 长期误差累积更稳定
- **对工作的价值**: ⭐⭐⭐⭐
- **摘要**: 学习型惯性里程计通常回归绝对位置，容易产生大的常数误差。本文改为估计 50ms 滑窗内的增量位移 Δp，通过数值积分重建轨迹。对比标准 MLP 和 KAN（可学习 B-spline 激活），KAN 参数量仅 8444（MLP 57859），最终累积漂移 9.61m vs MLP 17.23m，降低 44%。KAN 在 P50 和 P90 累积漂移上均更稳定，表明可学习 B-spline 激活在惯性里程计误差累积控制上有潜力。

### 7. [ISOPoT: Imaging Sonar Odometry by Point Tracking](https://arxiv.org/abs/2606.23006)
- **方向**: 📡 多传感器融合, 🎯 紧耦合前端
- **门槛**: CPU 实时（点跟踪 + 轻量优化）
- **核心 idea**: 声呐图像里程计用多帧点轨迹做对应表示，替代标准视觉关键点检测
- **关键点**: Aracati 2017 + 内部真实水下数据集；纯声呐和多传感器场景均超越 SOTA
- **对工作的价值**: ⭐⭐⭐
- **摘要**: 水下环境可靠导航是海洋机器人关键挑战。前视声呐在浑浊低能见度条件下提供大覆盖感知，但声呐图像噪声大、伪影多、缺乏语义结构，标准视觉方法表现差。ISOPoT 提出基于现代点跟踪技术的声呐里程计，使用多帧点轨迹作为主要对应表示，辅以轻量优化提升鲁棒性。在 Aracati 2017 数据集和内部真实水下数据集上评估，纯声呐和多传感器场景均超越之前 SOTA。

### 8. [MIL-LC: A Robust Magnetometer-Inertial-LiDAR Fusion Multimodal Localization Framework](https://arxiv.org/abs/2606.25796)
- **方向**: 📡 多传感器融合, 🔁 回环 / 位置识别
- **门槛**: CPU 实时（磁力计 + IMU + LiDAR 融合，滤波级别）
- **核心 idea**: 地磁场作为 LiDAR 几何退化的互补模态，无需额外基础设施
- **关键点**: 自研传感器套件；GNSS-denied、几何重复、纹理缺失场景验证；磁地图长期变化下仍鲁棒
- **对工作的价值**: ⭐⭐⭐
- **摘要**: GNSS-denied、几何重复或纹理缺失场景下定位仍是 AMR 部署的开放问题。单模态定位受限于传感器约束，多模态融合大多依赖几何/纹理特征或基础设施信标。地磁场定位不依赖几何/纹理、不需要额外基础设施，是有前景的互补模态。但现有研究仅在行人智能手机场景探索。MIL-LC 提出磁力计-惯性-LiDAR 融合多模态定位框架，配备自研传感器套件，在 LiDAR 几何退化或磁地图长期变化时仍提供可靠定位。

## 🟡 Tier-2: 需要 10-100 TOPS NPU（6 篇）

### 9. [BLENDS: Bayesian Learning-Enhanced Deep Smoothing for GNSS-Denied Environments](https://arxiv.org/abs/2606.22456)
- **方向**: 📡 多传感器融合, 🧠 SLAM + 深度学习
- **门槛**: Transformer 模型，需轻量 GPU/NPU 推理
- **核心 idea**: Transformer 增强 Bayesian 平滑，学习协方差自适应和状态修正，GNSS 中断时仍保持精度
- **关键点**: 四旋翼数据集 GNSS 中断验证；位置 RMSE 提升 25.6%；学习补偿 GNSS 常规定位与 RTK 真值间系统偏差
- **对工作的价值**: ⭐⭐⭐⭐⭐
- **摘要**: GNSS 中断期间保持精确导航是依赖低成本惯性传感器的自主系统的重大挑战。经典平滑方法（如 RTS smoother）利用中断前后测量值，但精度受限于常规 GNSS 测量。BLENDS 提出 Transformer 框架，增强 Bayesian 平滑的学习协方差自适应和状态修正。在四旋翼 GNSS 中断数据集上验证，位置 RMSE 提升 25.6%，降低估计不确定性。BLENDS 还学会了补偿常规 GNSS 定位与 RTK 真值间的系统偏差，精度超越常规 GNSS 测量本身。

### 10. [OrthoTrack: Continuous 6-DoF UAV Trajectory Estimation Anchored in Public Orthophotos](https://arxiv.org/abs/2606.25245)
- **方向**: 📍 主动 / 事件相机, 📡 多传感器融合
- **门槛**: 单 GPU 实时（ECCV 2026）；光流传播 + 正射影像匹配，Jetson 可行
- **核心 idea**: 用公开正射影像 + 地表模型做 map prior，关键帧匹配后光流传播到中间帧，无 GPS 实现绝对度量 6-DoF
- **关键点**: Training-free；MovingDrone 数据集（大规模 UAV + 密集 6-DoF 真值）；超越所有 baseline 含 oracle scale
- **对工作的价值**: ⭐⭐⭐⭐⭐
- **摘要**: 连续 6-DoF 位姿估计对自主 UAV 至关重要，但现有 VO/SLAM 累积漂移且只有相对 up-to-scale 轨迹，单帧地理定位丢弃时间连续性且太慢。OrthoTrack 是 training-free 系统，仅用公开正射影像和地表模型作为 map prior，将关键帧与正射影像匹配，通过地表模型提升到度量 3D，再用光流传播到中间帧，无需 GPS 或事后对齐即可在每帧产生绝对度量位姿。发布 MovingDrone 数据集，大规模真实世界基准上实时运行，大幅超越所有 baseline。

### 11. [UAV-MapFusion: RTK-Aligned Uncertainty-Aware Coarse-to-Fine Multi-Session UAV Mapping](https://arxiv.org/abs/2606.26928)
- **方向**: 📡 多传感器融合, 🗺️ 语义建图
- **门槛**: 因子图优化 + MOGP，Jetson 可跑（RA-L 2026）
- **核心 idea**: 多次飞行点云地图融合，RTK 时空对齐（DTW 估计时间偏移 + MOGP 恢复连续约束），统一不确定性因子图
- **关键点**: 场景图初始合并 → RTK 对齐 → 迭代平面因子精修；代码和数据集将开源
- **对工作的价值**: ⭐⭐⭐⭐⭐
- **摘要**: 大规模点云地图对机器人和空间智能任务至关重要。UAV 受限于续航和存储，单次飞行难以覆盖大场景。现有多 session 地图合并方法在 UAV 场景下难以同时抑制长程漂移和保持局部几何精度。UAV-MapFusion 提出不确定性感知多 session 点云地图合并和 coarse-to-fine 优化系统：场景图初始合并 → RTK 时空对齐模块（DTW 估计时间偏移，MOGP 在不完整采样和丢帧下恢复连续 RTK 约束）→ 统一不确定性因子图 → 迭代平面因子精修。真实数据集验证有效性，代码将开源。

### 12. [Commerge: Communication-Efficient, Robust, and Fast LiDAR Map Merging Framework](https://arxiv.org/abs/2606.25386)
- **方向**: 🌐 分布式 / 多机, ⚙️ 轻量化
- **门槛**: CPU 可跑（图论选择数据交换），通信量极低适合带宽受限多机
- **核心 idea**: 图论选择性数据交换，只传精心选择的扫描子集，通信量降低 5000 倍
- **关键点**: 三级级联优化（顺序重叠 → 传输均衡 → 几何感知最优）；7GB→1.3MB；5 公开 + 4 内部数据集
- **对工作的价值**: ⭐⭐⭐⭐
- **摘要**: 多机器人 LiDAR 地图合并需要交换大量传感器数据，通信成为瓶颈。Commerge 通过图论选择性数据交换实现带宽缩减：只有精心选择的扫描子集就足够鲁棒合并。形式化为交换图上的三级级联优化——顺序重叠、均衡传输成本、几何感知最优扫描子集。不同于传整扫描（GB 级）或朴素下采样，Commerge 仅交换 MB 级数据达到可比精度。在 5 个公开数据集和 4 个内部数据集（洞穴、行星模拟、室内、室外校园）上评估，数据交换减少 99.98%（7GB→1.3MB）。

### 13. [ADM-Fusion: Adaptive Deep Multi-Sensor Fusion for Robust Ego-Motion Estimation](https://arxiv.org/abs/2606.25111)
- **方向**: 📡 多传感器融合, 🧠 SLAM + 深度学习
- **门槛**: 深度学习模型（MoE 架构），需 NPU 推理
- **核心 idea**: 自适应传感器混合专家框架 + 内容感知路由，实时动态分配传感器权重
- **关键点**: 平移/旋转分支 + 跨任务注意力；CARLA-LOC 训练 → KITTI 微调 sim-to-real；退化条件鲁棒
- **对工作的价值**: ⭐⭐⭐
- **摘要**: 多传感器融合在多样和退化环境中，传感器可靠性快速波动时至关重要。不同模态失败方式不同，有效融合应自适应平衡互补线索而非固定权重。ADM-Fusion 提出端到端深度学习多传感器融合方法，采用自适应传感器混合专家框架和内容感知路由实时动态分配权重。分离平移和旋转分支，通过跨任务注意力保持任务特化同时信息共享。CARLA-LOC 仿真训练后 KITTI 真实数据微调，验证 sim-to-real 迁移。

### 14. [SkyJEPA: Learning Long-Horizon World Models for Zero-Shot Sim-to-Real Control of Quadrotors](https://arxiv.org/abs/2606.23444)
- **方向**: 🧠 SLAM + 深度学习, 🤖 SLAM + 大模型
- **门槛**: JEPA 潜空间模型 + 采样控制，嵌入式硬件实时部署（作者声明）
- **核心 idea**: JEPA 风格潜空间动力学模型 + 物理启发 prober，四旋翼零样本 sim-to-real 控制
- **关键点**: Yann LeCun 参与；开源 (github.com/arplaboratory/SkyJEPA)；室外闭环验证；结构化数据集生成 pipeline
- **对工作的价值**: ⭐⭐⭐⭐
- **摘要**: 敏捷飞行器在不确定下需要精确动力学模型。神经网络动力学模型能捕获复杂非线性，但现有预测方法的自回归 rollout 机制在长时域预测中误差累积。JEPA 在潜空间建模动力学，但之前主要用于运动学规划，高频控制探索有限。SkyJEPA 将 JEPA 引入四旋翼实时控制，结合潜空间动力学模型和物理启发 prober（将冻结潜变量映射到可解释状态），实现物理约束的长时域预测。与采样最优控制结合，在嵌入式硬件实时运行。结构化数据集生成 pipeline 减少真实数据采集需求。开环和室外闭环实验验证零样本 sim-to-real 和跨条件泛化。

## 🔴 Tier-3: 算力超限但技术追踪（4 篇）

### 15. [Spectral GS-SLAM: Observability-Aware, Degeneracy-Robust Tracking for Real-Time 3D Gaussian Splatting SLAM](https://arxiv.org/abs/2606.21258)
- **方向**: 🧠 SLAM + 深度学习, 🗺️ 语义建图
- **门槛**: 3DGS-SLAM 需中等 GPU（TUM RGB-D 40 FPS），但 3DGS 渲染对无人机嵌入式端偏重
- **核心 idea**: ICP + 特征约束互补，谱方法自适应补偿欠约束方向，高斯平面性加权
- **关键点**: IROS 2026；TUM RGB-D 40.14 FPS；结构/纹理缺失场景轨迹完整性保持
- **对工作的价值**: ⭐⭐⭐
- **摘要**: 3DGS-SLAM 用特征匹配或 ICP 做跟踪避免密集光度优化，但特征匹配在无纹理场景失败，ICP 在无结构/几何退化场景因病态优化而挣扎。Spectral GS-SLAM 集成 ICP 与互补特征约束，通过自适应补偿退化场景的欠约束方向来缓解数值不稳定性，不干扰共享高斯表示。引入高斯感知平面性加权机制利用 3D 高斯协方差结构表征场景几何。TUM RGB-D 上 40.14 FPS 实时，退化场景保持轨迹完整性。

### 16. [Pocket-SLAM: Rendering-Area-Aware Pruning for Memory-Efficient 3DGS-SLAM](https://arxiv.org/abs/2606.24796)
- **方向**: 🧠 SLAM + 深度学习, ⚙️ 轻量化
- **门槛**: 3DGS-SLAM，ICRA 2026；内存优化但对无人机嵌入式仍偏重
- **核心 idea**: 渲染面积感知剪枝，按高斯对有效渲染区域的贡献选择性移除，而非仅靠 opacity/gradient
- **关键点**: EuRoC + KITTI；内存降低 60%+，FPS 提升 2 倍+；代码开源
- **对工作的价值**: ⭐⭐⭐
- **摘要**: 3DGS-SLAM 在大规模场景中内存消耗随高斯点累积持续增长。Pocket-SLAM 提出渲染面积感知剪枝策略，按高斯对有效渲染区域的贡献选择性移除，而非仅靠 opacity/gradient 等高斯级启发式。EuRoC 和 KITTI 上内存降低 60%+，FPS 提升 2 倍+，保持定位建图精度。

### 17. [DSP-SLAM++: A Unified Framework for Multi-Class, High-Fidelity Object SLAM in the Wild](https://arxiv.org/abs/2606.25953)
- **方向**: 🗺️ 语义建图, 🧠 SLAM + 深度学习
- **门槛**: 多类物体高保真重建，需 GPU
- **核心 idea**: 异步建图 pipeline + 鱼眼-LiDAR 融合，实时多类高保真物体 SLAM
- **关键点**: 25Hz 多类数据集；最大物体处理延迟降低 70%；代码开源
- **对工作的价值**: ⭐⭐⭐
- **摘要**: 物体感知 SLAM 需在实时性、多类支持和高保真物体模型间权衡。DSP-SLAM++ 扩展 DSP-SLAM，异步建图 pipeline 实现实时性，鱼眼-LiDAR 传感器融合适配。多类数据集上生成细粒度几何完整形状，最大物体处理延迟降低 70%，25Hz 实时。

### 18. [RoboAtlas: Contextual Active SLAM](https://arxiv.org/abs/2606.26046)
- **方向**: 🤖 SLAM + 大模型, 🗺️ 语义建图
- **门槛**: GPT-4o 推理 + 3D 语义建图（OpenRoboVox），算力超无人机红线
- **核心 idea**: 上下文多臂赌博机在几何探索和语义推理间自适应切换，GPT-4o + 大规模 3D 语义地图
- **关键点**: GOAT-Bench Val Unseen SOTA（90.6% SR）；Qwen2.5-VL-7B 也达 88.8%；1800m² 真实环境 100% 成功率
- **对工作的价值**: ⭐⭐⭐
- **摘要**: RoboAtlas 提出上下文 Active SLAM 框架，通过上下文多臂赌博机自适应平衡几何探索和语义推理。集成前沿探索、全局语义地图推理和 egocentric VLM 推理。GOAT-Bench 上 90.6% 成功率（GPT-4o），Qwen2.5-VL-7B 也达 88.8%，超过所有用 GPT-4o 的 baseline。1800m² 真实环境 30k 语义实例 100% 任务成功率。
