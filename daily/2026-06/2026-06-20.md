# 2026-06-20 无人机 SLAM 论文阅读 · 本周复盘

这周整体还不错，有几篇跟多旋翼无人机状态估计直接相关的，还有一篇直接用电机转速替代 IMU 做预积分的，思路挺新鲜。下面按三档整理。

## 🟢 Tier-1: CPU 实时就能跑（9 篇）

### 1. [Motor Angular Speed Preintegration for Multirotor UAV State Estimation](https://arxiv.org/abs/2606.19929)
- **方向**: 📡 多传感器融合, 🎯 紧耦合前端
- **门槛**: CPU 实时，因子图优化 + LiDAR 里程计
- **核心 idea**: 用多旋翼的电机角速度做预积分，替代/补充 IMU 来做状态传播。电机转速推算出的加速度比被桨叶振动污染的 IMU 数据更干净，直接在因子图里加一个 motor speed factor。
- **关键点**: 开源了 MAS-LO 算法；相比 LIO-SAM 位置精度提升 28%、速度精度提升 65%、测量延迟降低 14%；来自捷克 MRS 组（做无人机 swarm 很猛的组）。
- **对工作的价值**: ⭐⭐⭐⭐⭐ — 直接针对多旋翼 UAV 状态估计，思路新颖，开源可复现。电机转速传感器本身就在飞控上，零额外硬件成本。
- **摘要**: 多旋翼的 IMU 被螺旋桨振动严重污染，常规做法是低通滤波但会引入延迟。这篇提出用电机角速度推算机身加速度——物理上螺旋桨推力跟转速平方成正比，从转速反推加速度在理论上比振动污染的 IMU 更干净。他们把电机转速预积分做成一个 factor，跟 LiDAR 里程计紧耦合，实验证明精度和延迟都优于 LIO-SAM。

### 2. [FAST-LIVGO: A Degeneracy-Robust LiDAR-Inertial-Visual-GNSS Fusion Odometry](https://arxiv.org/abs/2606.19190)
- **方向**: 📡 多传感器融合, 🛰️ 视觉+GNSS, 🎯 紧耦合后端
- **门槛**: CPU 实时，ESIKF 框架
- **核心 idea**: 把 LiDAR + IMU + 视觉 + GNSS 四种传感器紧耦合到一个 ESIKF 里，关键创新是退化感知的双模式 outlier rejection——LIVO 退化时自动切到 GNSS 主导恢复。
- **关键点**: IROS 2026；DTW 做时空对齐应对高动态；多普勒频移 + TDCP 提供毫米级相对约束；在 20m/s 固定翼 UAV 数据集上验证。
- **对工作的价值**: ⭐⭐⭐⭐⭐ — 四传感器紧耦合，固定翼 UAV 实测，退化场景鲁棒性策略直接可用。
- **摘要**: LIVO 系统在退化场景（长廊、开阔地、弱纹理）会漂移甚至挂掉，GNSS 融合通常依赖里程计做 outlier rejection，一旦里程计本身退化就一起崩。这篇用退化感知的双模式切换：里程计健康时用 LIVO 做先验去筛 GNSS outlier，退化时反过来用 GNSS 约束恢复里程计。加入了多普勒频移和 TDCP 观测模型，在 20m/s 固定翼上验证了效果。

### 3. [Observability and Consistency Analysis for Visual-Inertial Navigation with Anchored Feature Parameterizations](https://arxiv.org/abs/2606.19307)
- **方向**: 🎯 紧耦合前端 / 后端
- **门槛**: CPU 实时，纯理论分析 + 仿真验证
- **核心 idea**: 分析了 anchored feature 参数化（特征点相对于 anchor frame 而非全局坐标系）对 VINS 可观测性和一致性的影响。结论：用 anchored 表示时，不可观子空间不依赖路标点估计值，天然改善一致性。
- **关键点**: IROS 2026；TUM-VI 数据集实测；不需要额外 trick 就能达到跟加了 consistency enforcement 的全局参数化相近的性能。
- **对工作的价值**: ⭐⭐⭐⭐ — VINS 理论基础，anchored 参数化在工程上很容易实现（本质就是相对坐标），对滤波器一致性有帮助。
- **摘要**: 滤波 VINS 的不可观子空间通常依赖当前导航状态估计，导致不一致。这篇发现如果路标点用 anchored 参数化（相对某个 anchor frame），不可观子空间不再依赖路标估计值，天然改善一致性。仿真和 TUM-VI 实验证实了这一点。对做滤波器 VIO 的人来说，这个结论意味着选对参数化方式可能比加一堆 FEJ/OC 的 trick 更有效。

### 4. [Sensor Configuration Matters: A Systematic Evaluation of Multimodal SLAM on Quadruped Robots](https://arxiv.org/abs/2606.19067)
- **方向**: 📡 多传感器融合
- **门槛**: CPU 实时（评测论文，不涉及新算法）
- **核心 idea**: 系统评估了相机模态（单目/双目/RGB-D）、快门类型（全局/卷帘）、IMU 等级对足式机器人 SLAM 的影响。结论：双目 >> 单目，全局快门 >> 卷帘快门，IMU 在剧烈运动下反而可能拖累纯视觉方法。
- **关键点**: GrandTour 数据集（ANYmal D 四足）；结论对无人机也有参考价值——无人机的高频振动跟四足的足端冲击类似。
- **对工作的价值**: ⭐⭐⭐⭐ — 传感器选型指南。无人机上同样面临振动问题，IMU 是否会拖累视觉前端的结论值得关注。
- **摘要**: 在四足机器人上跑了一圈 SOTA SLAM 系统，发现硬件配置的影响比想象中大得多：卷帘快门在剧烈运动下直接导致跟踪失败，标准 IMU 融合在纯视觉框架里反而降低性能。这些结论对无人机传感器选型有直接参考——我们的震动环境跟四足机器人很接近。

### 5. [FD-SLAM: Fast Dense Radar-Inertial SLAM with Frequency-Domain Loop Closure and Pose Graph Optimization](https://arxiv.org/abs/2606.15491)
- **方向**: 📡 多传感器融合, 🔁 回环 / 位置识别
- **门槛**: CPU 实时（论文明确说前端超雷达采样率跑在 CPU 上）
- **核心 idea**: 雷达-惯性 SLAM，用频域极坐标描述子做回环检测 + 多阶段验证 pipeline。频域方法天然抗雷达噪声。
- **关键点**: 前端 CPU-only；回环检测后台并行；在公开数据集上跟 SOTA 雷达 SLAM 持平。
- **对工作的价值**: ⭐⭐⭐ — 雷达在无人机上不常见（太重），但频域回环检测的思路对视觉回环也有启发。如果未来考虑雷达-视觉融合降落可以翻翻。
- **摘要**: 扫描雷达噪声大、帧率低，但全天候可用。这篇用频域极坐标描述子做回环检索——频域表示对雷达散斑噪声不敏感。回环验证 pipeline 包含时域滤波、相位相关筛选、扫描对齐相似度和几何一致性检查四步，CPU 上实时跑。

### 6. [RICH-SLAM: Radar SLAM with Incremental and Continuous Hilbert Mapping](https://arxiv.org/abs/2606.17534)
- **方向**: 📡 多传感器融合, 🗺️ 语义建图
- **门槛**: CPU 实时（粒子滤波 + 希尔伯特空间 GP）
- **核心 idea**: Rao-Blackwellized 粒子滤波做雷达 SLAM，用增量希尔伯特空间降秩 GP 把稀疏雷达测量变成连续占栅格地图。还引入后验感知的粒子权重，用地图参数的完整后验分布做更鲁棒的似然评估。
- **关键点**: ColoRadar 数据集验证；连续地图支持不确定性感知的路径规划。
- **对工作的价值**: ⭐⭐⭐ — 雷达在无人机上不常见，但 RB-PF + GP 连续建图的方法论可以迁移到视觉/激光。
- **摘要**: 雷达测量稀疏且噪声大，传统 SLAM 很难建出稠密连续的地图。这篇用希尔伯特空间的降秩 GP，从稀疏雷达点构建连续占栅格地图，同时保留不确定性信息。粒子权重计算时考虑了地图后验分布而不仅仅是点估计，更鲁棒。

### 7. [A High-accuracy Event-based Underwater SLAM System](https://arxiv.org/abs/2606.18951)
- **方向**: 📍 事件相机
- **门槛**: CPU 实时（贝叶斯优化找最优参数在线运行）
- **核心 idea**: 第一个高精度事件相机水下双目 SLAM。解决事件相机在水下的 Time Surface 质量不稳定问题——用结构张量 coherence 评估 TS 质量，贝叶斯优化找最优 TS 参数。
- **关键点**: 贡献了 UWE 数据集（水下事件相机）；代码和数据将开源。
- **对工作的价值**: ⭐⭐⭐ — 事件相机在无人机快速机动场景有潜力，水下挑战（动态光照、弱纹理）跟无人机低空场景有相似之处。
- **摘要**: 事件相机在水下 Time Surface 成像质量随速度剧烈波动，传统方法频繁挂掉。这篇用结构张量 coherence 量化 TS 质量，通过解耦优化（初始化前 BO 找先验 + 跟踪阶段在线局部搜索）保证实时 TS 质量。还引入了 prior disparity 辅助匹配和"最新观测优先"三角化。

### 8. [Route-Constrained Robust Fusion Estimation for MEMS/GNSS Integrated Navigation of Unmanned Ground Vehicles in GNSS Degraded Environments](https://arxiv.org/abs/2606.19687)
- **方向**: 🛰️ 视觉+GNSS
- **门槛**: CPU 实时（EKF + 二维刚体变换）
- **核心 idea**: GNSS 丢失时，把航位推算轨迹跟高精地图的路线做 2D 刚体对齐，生成伪位置观测注入 EKF 更新。用路线级约束抑制位置漂移。
- **关键点**: ICRA 2026 Workshop；长隧道、多段隧道、弯曲隧道三种场景验证；工程化策略完备（触发控制、匹配质量验证、偏移补偿等）。
- **对工作的价值**: ⭐⭐⭐ — 虽然面向地面车辆，但"先验路线约束注入 EKF"的思路在无人机航线飞行中可以直接借鉴。
- **摘要**: MEMS IMU + GNSS 在隧道里位置漂移不可控。这篇把航推轨迹跟高精地图路线做 2D 刚体对齐，得到一个路线参考位置，作为伪观测注入 EKF。同时加了触发控制、匹配质量验证等工程策略防止错误约束。三个隧道场景验证有效。

### 9. [BIM-Loc: BIM-Integrated Discrepancy-Aware LiDAR-based Indoor Localization](https://arxiv.org/abs/2606.14237)
- **方向**: 🗺️ 语义建图
- **门槛**: CPU 实时（pose graph + 贝叶斯推断）
- **核心 idea**: 用 BIM 模型（建筑设计阶段的 3D 模型）作为先验地图，激光雷达在线定位的同时检测真实环境跟 BIM 的差异。多射线投射做数据关联 + 层次贝叶斯推断做差异检测。
- **关键点**: IJRR 已接收；在仿真和真实环境中验证。
- **对工作的价值**: ⭐⭐ — BIM 本身跟无人机场景关系不大，但"在线定位+地图差异检测"的思路对无人机长期自主飞行中的地图更新有启发。
- **摘要**: 室内环境经常变化，预建地图容易过时。这篇直接把设计阶段的 BIM 模型当先验地图，在线定位的同时检测环境变化。用多射线投射做高效数据关联，层次贝叶斯做差异检测——从像素级更新传播到结构级。

## 🟡 Tier-2: 需要 10-100 TOPS NPU / Jetson 级别（8 篇）

### 10. [MMD-SLAM: Structure-Enhanced Multi-Meta Gaussian Distribution-Guided Visual SLAM](https://arxiv.org/abs/2606.19874)
- **方向**: 🧠 SLAM + 深度学习, 🗺️ 稠密重建
- **门槛**: Jetson Orin 级别（3DGS 渲染 + 优化）
- **核心 idea**: 把 Atlanta World 假设（城市场景中平面多沿正交方向）引入 3DGS-SLAM：用点线融合做位姿优化，多 Meta Gaussian 显式编码结构先验，Gaussian 演化策略自适应场景几何。
- **关键点**: ICRA 2026；ScanNet ATE RMSE 降低 48.56%；Replica PSNR 提升 5.71%。
- **对工作的价值**: ⭐⭐⭐⭐ — 3DGS-SLAM 是当前热点，这篇的结构先验思路对无人机建图有参考价值。但目前 3DGS 渲染在 Jetson 上还比较吃力。
- **摘要**: 现有 3DGS-SLAM 没利用好场景结构信息，渲染质量受限。这篇引入 Atlanta World 假设——城市环境中的墙面和地面多沿正交方向——用点线融合和 Multi-Meta Gaussian 显式编码结构先验。在 ScanNet 和 Replica 上取得了 SOTA。

### 11. [G2IA: Geometry-Guided Instance-Aware Retrieval and Refinement for Cross-Modal Place Recognition](https://arxiv.org/abs/2606.15287)
- **方向**: 🔁 回环 / 位置识别
- **门槛**: Jetson 级别（VGGT 视觉几何先验）
- **核心 idea**: 跨模态位置识别（RGB 图像 → LiDAR 地图），用 VGGT 提取视觉几何先验 + 实例特征做检索，再用局部实例形状和空间布局做重排序验证。
- **关键点**: 跨数据集泛化能力强；对无人机用视觉在 LiDAR 先验地图中定位有直接价值。
- **对工作的价值**: ⭐⭐⭐⭐ — 无人机在预建 LiDAR 地图中的视觉定位，这个方向对自主返航/降落很有意义。
- **摘要**: 图像到点云的位置识别有两个难点：模态鸿沟（RGB 外观 vs 稀疏几何）和感知混淆（相似的道路/建筑结构）。这篇用 VGGT 提取的几何先验 + 实例特征构建兼容 LiDAR 地图的 place descriptor，重排序阶段验证局部实例形状和空间布局是否跨模态一致。

### 12. [Hardware- and Vision-in-the-Loop Validation of Deep Monocular Pose Estimation for Autonomous Maritime UAV Flight](https://arxiv.org/abs/2606.19176)
- **方向**: 🧠 SLAM + 深度学习, 📡 多传感器融合
- **门槛**: Jetson 级别（transformer 单目位姿估计 + Kalman 滤波）
- **核心 idea**: 在室内用 photorealistic 渲染模拟海上环境，做硬件在环验证——深度学习单目位姿估计跑在真实嵌入式计算平台上，延迟 Kalman 滤波融合 IMU，实现全自主起飞/跟踪/降落。
- **关键点**: 捕获了真实嵌入式计算延迟、异步更新等纯仿真看不到的问题；几何控制器做闭环。
- **对工作的价值**: ⭐⭐⭐⭐ — 海上 UAV 降落是位姿估计的硬核场景。硬件在环验证方法论本身对无人机 SLAM 开发很有借鉴——在室内就能验证真实嵌入平台上的性能。
- **摘要**: 海上无人机降落需要可靠的视觉相对位姿估计，但出海测试成本高且风险大。这篇在室内搭建了一套硬件在环系统：photorealistic 渲染海上视角 → 真实嵌入平台跑 transformer 位姿估计 → 延迟 Kalman 滤波融合 IMU → 几何控制器闭环。关键是捕获了嵌入式计算延迟和异步更新的真实影响。

### 13. [Aerial-ground LiDAR place recognition with patch-level self-supervised learning and expanded reciprocal re-ranking](https://arxiv.org/abs/2606.18583)
- **方向**: 🔁 回环 / 位置识别
- **门槛**: Jetson 级别（自监督学习网络）
- **核心 idea**: 空地 LiDAR 位置识别——用机载 ALS 数据做先验地图，地面 LiDAR 查询定位。多尺度 patch 级自监督学习 + 扩展互惠重排序。
- **关键点**: CS-Urban-Scenes 上 Recall@1 提升 9.8%；ER 重排序无需额外训练再提 10.2%。
- **对工作的价值**: ⭐⭐⭐⭐ — 空地跨视角位置识别对无人机在预建地图中定位非常实用。ALS 数据获取成本在降低（越来越多城市有公开数据）。
- **摘要**: ALS（机载激光扫描）覆盖全、视角好，但跟地面 LiDAR 之间存在域鸿沟。这篇用 patch 级自监督学习——利用相邻 patch 语义相似性——提升跨域特征判别力。重排序阶段利用 ALS 的空间结构分布做扩展互惠重排序，无需额外训练大幅提升 Recall。

### 14. [Spatially Stratified Distillation for Heterogeneous Radar Place Recognition](https://arxiv.org/abs/2606.18687)
- **方向**: 🔁 回环 / 位置识别, ⚙️ 轻量化/蒸馏
- **门槛**: Jetson 级别（知识蒸馏）
- **核心 idea**: 异构雷达位置识别——便宜的 4D 车载雷达（稀疏）vs 高端旋转雷达（稠密）之间的知识蒸馏。关键创新：只在两个雷达都有回波的区域做强对齐，稀疏区域降低蒸馏权重，避免强行学习噪声。
- **关键点**: ICRA 2026 Workshop；HeRCULES 数据集 SOTA。
- **对工作的价值**: ⭐⭐⭐ — 知识蒸馏的"按区域对齐"策略对无人机上轻量化模型的训练有启发。4D 雷达在无人机上不实用，但蒸馏方法论值得看。
- **摘要**: 4D 雷达便宜但极其稀疏，做位置识别时跟高端雷达地图匹配困难。之前的方法统一投影做蒸馏，在多场景环境下性能下降。这篇提出空间分层蒸馏：只在两雷达重叠区域做强对齐，稀疏区域降权。蒸馏策略本身对无人机上模型压缩有参考。

### 15. [SGM-SLAM: Scene Graph Matching for Data-Efficient Distributed SLAM](https://arxiv.org/abs/2606.16881)
- **方向**: 🌐 分布式 / 多机
- **门槛**: Jetson 级别（RGB-LiDAR 语义分割 + 场景图）
- **核心 idea**: 多机器人分布式 SLAM，用场景图匹配（物体标签 + 质心）而非特征级匹配来做机器人间的测量约束。通信效率高——只交换物体标签和质心。
- **关键点**: 仿真 + 真实四足机器人室内外验证；首次只用物体标签和质心做场景图匹配。
- **对工作的价值**: ⭐⭐⭐ — 多无人机协同 SLAM 的场景下，通信带宽是瓶颈。只用物体标签+质心做匹配的思路大幅降低通信量，值得借鉴。
- **摘要**: 多机器人 SLAM 的传统做法是交换特征点描述子，通信量大。这篇用语义分割出物体，每个物体只传标签+质心，场景图匹配后做位姿图优化。室内外四足机器人实验验证了效果。

### 16. [SplatlessDF: Continuous Distance Field Mapping with Non-Splatting Gaussians](https://arxiv.org/abs/2606.13990)
- **方向**: 🗺️ 稠密重建
- **门槛**: Jetson 级别（Gaussian 优化）
- **核心 idea**: 用各向异性高斯元做连续距离场建图（不是渲染！），使下游导航任务可以直接查距离和梯度。可单独跑（纯 DF）或跟 2DGS 联合。
- **关键点**: 高斯表示不仅用于渲染，还能做空间场建图；纯 DF 模式效率高。
- **对工作的价值**: ⭐⭐⭐ — 距离场对无人机避障规划很有用。高斯表示比 TSDF/ESDF 更紧凑，值得关注这个方向。
- **摘要**: 3DGS 的成功启发了新的场景表示方法。这篇从空间角度（而非光度角度）用各向异性高斯做连续距离场，使导航可以直接查距离和梯度。两种模式：纯 DF（快）和 DF+2DGS 联合（能渲染+建图）。

### 17. [GAIT: Legged Robot Proprioceptive State Estimation with Attention over Inertial-Leg Tokens](https://arxiv.org/abs/2606.14160)
- **方向**: 🧠 SLAM + 深度学习, 📡 多传感器融合
- **门槛**: Jetson 级别（轻量 attention 网络）
- **核心 idea**: 把 IMU 和每条腿的测量分别 token 化，用 attention 学习每个测量的相对重要性——根据接触状态自动加权。不依赖显式接触检测。
- **关键点**: Unitree Go1 实测，未见过的步态和地形上优于 SOTA。
- **对工作的价值**: ⭐⭐ — 面向四足，但 tokenization + attention 做多传感器自适应的思路对无人机多传感器融合有启发。
- **摘要**: 传统学习方法把所有传感器拼成一个向量输入，没法根据接触状态动态调整权重。这篇把每个传感器的测量单独 token 化，attention 自动学习重要性权重——腿接触地面时运动学测量更可信，离地时降权。这个传感器自适应的思路对无人机也适用。

### 18. [LIT-GS: LiDAR-Inertial-Thermal Gaussian Splatting for Illumination-Robust Mapping](https://arxiv.org/abs/2606.20424)
- **方向**: 📡 多传感器融合, 🗺️ 稠密重建
- **门槛**: GPU 集群（3DGS + 多模态融合）
- **核心 idea**: LiDAR + IMU + 热成像做 Gaussian Splatting 建图，在光照变化/弱纹理场景下比纯 RGB 的 LIV-GS 更鲁棒。LiDAR 点面残差约束 BA 和高斯优化。
- **IROS 2026**；热成像+LiDAR+IMU 的组合在无人机夜间巡检场景有价值，但目前 3DGS 整体在嵌入端还跑不动。
- **摘要**: 把热成像引入 Gaussian Splatting 建图，解决 RGB 在光照变化场景不可靠的问题。LiDAR 提供的平面几何约束同时用于 BA 和高斯优化，减轻热成像低对比度导致的表面增厚和结构漂移。

### 19. [Splaxel: Efficient Distributed Training of 3D Gaussian Splatting for Large-scale Scene Reconstruction via Pixel-level Communication](https://arxiv.org/abs/2606.18588)
- **方向**: 🌐 分布式, 🗺️ 稠密重建
- **门槛**: GPU 集群（分布式 3DGS 训练）
- **核心 idea**: 分布式 3DGS 训练，像素级通信降低同步开销。大规模场景重建方向，跟实时 SLAM 关系不大但 3DGS 训练效率提升值得追踪。
- **摘要**: 大规模 3DGS 训练需要多 GPU，传统参数同步开销大。这篇用像素级通信降低同步成本，提升分布式训练效率。

### 20. [Occupancy-Grounded Room Segmentation for Hierarchical 3D Scene Graphs](https://arxiv.org/abs/2606.13727)
- **方向**: 🗺️ 语义建图
- **门槛**: 取决于后端（论文是方法框架）
- **核心 idea**: 用占栅格做房间分割，构建层次化 3D 场景图。对无人机语义导航的长期方向有参考。
- **摘要**: 3D 场景图是机器人长期自主的环境表示。这篇提出基于占栅格的房间分割方法，构建层次化场景图。不是 SLAM 方法本身，但对建图表示有启发。
