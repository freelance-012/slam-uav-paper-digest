# 2026-06-14 第 0 期 · 无人机 SLAM 经典方案基线盘点

最近在系统梳理一遍无人机上能跑得起来的 SLAM 方案，挑出这些已经被广泛验证、口碑稳定的"老兵"作为后续新论文对比的基线。

后面读新论文时，会拿它们在 EuRoC / TUM / KITTI 上的指标做参考，**也重点看作者有没有把这些方案跑在真机或硬件在环上**——纯数据集刷点的论文先放一边。

---

## 怎么分类

按"算力"和"传感器"两个维度切：

| 算力档 | 含义 |
|:---|:---|
| 🟢 Tier-1 | CPU 实时，板载 NUC / TX2 那种级别就能跑 |
| 🟡 Tier-2 | 需要 10-100 TOPS 的 NPU / 嵌入式 GPU（Orin / Xavier） |
| 🔴 Tier-3 | 算力超限，仅做趋势跟踪 |

下面按"纯视觉 → 视觉惯导 → 激光惯导 → 多传感器融合"的顺序展开。

---

## 一、纯视觉方案

### 🟢 ORB-SLAM3

- **核心思路**：ORB 特征 + 三线程（Tracking / Local Mapping / Loop & Map Merging）+ Atlas 多子地图。**视觉**和 **视觉-惯性**两套都吃。
- **优势**
  - 精度高、回环稳健，地图可复用
  - 代码工程化极好，开源社区文档全、二次开发友好
  - 支持单目/双目/RGBD，IMU 紧耦合版本也是工业界 benchmark
- **劣势 / 不 work 的场景**
  - **特征点法天生怕弱纹理**：白墙、走廊、纯色天花板直接挂
  - 动态场景（人车混杂）会大量误匹配，需要外加动态滤除
  - 计算量不低，**单目纯 CPU 大约 20-30 FPS**（i7），TX2 上能掉到 15 FPS
  - 初始化对运动有要求，纯旋转或静止启动容易失败
- **机载可行性**：✅ 但要降分辨率（VGA）+ 关掉冗余回环，能稳在 25-30 FPS
- **工程坑**：DBoW2 词袋加载耗时，ROS 启动建议异步预加载
- 仓库：https://github.com/UZ-SLAMLab/ORB_SLAM3

### 🟢 SVO / SVO2

- **核心思路**：直接法（光度误差）+ 半稠密深度估计，**没有显式回环**。SVO2 加入了 IMU 紧耦合。
- **优势**
  - 速度极快，**纯 SVO 在 i7 上能跑 100+ FPS**
  - 适合前向高速场景，FPV 无人机行业用得多
  - 对弱纹理比特征法鲁棒（直接法看像素灰度）
- **劣势 / 不 work 的场景**
  - **怕光照突变**：进出隧道、太阳眩光直接飘
  - 没有回环，长期运行会累积漂移
  - 单目尺度要靠 IMU 或外部给
- **机载可行性**：✅ 高速场景首选，但需要前端做曝光控制
- 仓库：https://github.com/uzh-rpg/rpg_svo / svo_pro

### 🟢 OV²SLAM

- **核心思路**：轻量化纯视觉，**有回环**（DBoW2）。代码量小、依赖少（OpenCV + Eigen），主打嵌入式。
- **优势**
  - 单目/双目/RGBD 通吃，**CPU 上能稳 30+ FPS**（单目）
  - 接口干净，集成进 ROS 很顺
  - 论文里在手机端都能跑
- **劣势 / 不 work 的场景**
  - 没有 IMU 紧耦合，**机动性强时尺度会飘**
  - 社区相比 ORB-SLAM3 小很多，遇到坑得自己啃
- **机载可行性**：✅ 算力紧张时的备选，但建议加 IMU 用 OpenVINS
- 仓库：https://github.com/ov2slam/ov2slam

## 二、视觉-惯导融合

### 🟢 VINS-Mono / VINS-Fusion

- **核心思路**：LK 光流前端 + 滑动窗口紧耦合优化 + 回环（DBoW2）。**滑动窗口 + IMU 预积分**是教科书范本。
- **优势**
  - 精度与 ORB-SLAM3 相当，但**前端更快**（光流比 ORB 提取便宜）
  - 紧耦合 IMU 让快速机动下也不挂
  - 在线外参标定、在线时间戳标定是杀手锏，**真机部署不用提前离线校准**
  - **i7 单目 30+ FPS**，TX2 上也能 20 FPS
- **劣势 / 不 work 的场景**
  - 光流法**怕运动模糊**（高速飞行时易丢点）
  - 长时间运行回环才拉得回漂移，无回环场景要靠 GPS 辅助
  - 鱼眼/广角相机的 distortion 要小心，初始化对畸变敏感
- **机载可行性**：✅ 业界事实标准之一，**很多无人机厂商直接拿 VINS 改**
- **工程坑**：feature_tracker 的并行化（OpenMP）开了才有 30 FPS
- 仓库：https://github.com/HKUST-Aerial-Robotics/VINS-Fusion

### 🟢 OpenVINS

- **核心思路**：MSCKF 风格的**滑动窗口滤波**（基于 EKF/ESKF）+ MSCKF + SLAM 混合特征。
- **优势**
  - **代码工程化极好**，模块化清晰，研究界和工业界都在用
  - 支持多相机、IMU 紧耦合、可视化工具完善
  - 论文里给的 ARM 平台数据，**TX2 跑 30 FPS 没问题**
- **劣势 / 不 work 的场景**
  - 基于 EKF，**对异常观测不够鲁棒**（ORB-SLAM3 那种 outlier rejection 更强）
  - 回环偏弱，长期大场景需要外接 RTK
- **机载可行性**：✅ VINS-Fusion 的强有力替代
- 仓库：https://github.com/rpng/open_vins

### 🟢 MSCKF（经典算法）

- **核心思路**：Mourikis 2007 的**多状态约束卡尔曼滤波**，把同一特征跨多帧的几何约束作为 EKF 更新。
- **优势**
  - 不需要保存特征到地图，**内存极小**（早期火星车就是这套思路）
  - 对快速运动友好（特征在滑动窗口内被几何消元）
  - 算力极低，**Cortex-A 系列都能跑**
- **劣势 / 不 work 的场景**
  - 纯滤波方法**精度通常比优化法低 10-30%**
  - 没有回环，大场景累积漂移严重
  - 现成可用的工业级 C++ 实现不多（ROVIO、OpenVINS 算是继承者）
- **机载可行性**：✅ 算力极端紧张时考虑，正常情况下 OpenVINS 更好

### 🟢 ROVIO

- **核心思路**：EKF + **图像块（patch）的光度误差**作为观测，IMU 做预测。
- **优势**
  - 视觉前端极轻量，**Pixhawk + 树莓派级别都能跑**
  - 对纹理变化比纯光流鲁棒（多层级 patch）
  - 苏黎世联邦理工出品，工程细致
- **劣势 / 不 work 的场景**
  - 滤波法精度的天花板
  - 参数调起来麻烦（process noise、patch 选取）
  - 社区活跃度下降
- **机载可行性**：✅ 微型无人机首选，但建议先评估 OpenVINS
- 仓库：https://github.com/ethz-asl/rovio

### 🟢 OKVIS

- **核心思路**：**基于关键帧的紧耦合优化**（VINS 之前的事实标准），前端 BRISK 特征。
- **优势**
  - 优化法精度天花板，**比 ROVIO 这种 EKF 准一截**
  - IMU 预积分的工业级实现
- **劣势 / 不 work 的场景**
  - 维护基本停止（2018 后基本没更新）
  - 文档比 VINS-Mono 差一截
  - 实时性**比 VINS 略差**（优化法天然计算量大）
- **机载可行性**：⚠️ 仍可用，但 VINS-Mono / OpenVINS 是更现代的选择
- 仓库：https://github.com/ethz-asl/okvis

### 🟢 DM-VIO

- **核心思路**：**延迟边缘化 + TSDF 视觉因子**。把当前帧延迟一帧再参与优化，配合 IMU 边缘化。
- **优势**
  - EuRoC 上**精度 SOTA 之一**
  - 对初始化不友好场景（纯旋转、慢速）有专门处理
  - IMU 偏置估计稳健
- **劣势 / 不 work 的场景**
  - **计算量比 VINS 大 30-50%**，TX2 上要优化
  - 代码复杂度高，**改起来不容易**
- **机载可行性**：⚠️ 精度优先的科研场景；机载要裁剪
- 仓库：https://github.com/lukasvst/dm-vio

### 🟢 BASALT

- **核心思路**：**视觉-惯导 + 视觉线段**（带语义）。ORB + LSD 线段混合特征。
- **优势**
  - 在人造结构（建筑、室内）下，**线段特征比点特征更稳**
  - 工程化好，配套可视化工具
  - TUM 出品
- **劣势 / 不 work 的场景**
  - 户外自然场景线段少，**优势发挥不出来**
  - 比 VINS 慢一些
- **机载可行性**：✅ 城市、桥梁、建筑巡检场景
- 仓库：https://gitlab.com/VladyslavUsenko/basalt

## 三、激光雷达-惯导

### 🟡 LOAM / A-LOAM

- **核心思路**：**点云曲率提取特征点**（edge + planar）+ 帧到帧 + 帧到地图的两阶段匹配。
- **优势**
  - LOAM 是 LiDAR SLAM 的**祖师爷**，概念经典
  - 精度高、原理清晰
  - **A-LOAM**（简化版）代码清爽，是入门 LiDAR SLAM 的好材料
- **劣势 / 不 work 的场景**
  - **没有 IMU 紧耦合**，快速运动下点云畸变严重
  - 没有回环，大场景累积漂移
  - 原始 LOAM 依赖 ROS1 旧接口，编译容易出坑
- **机载可行性**：⚠️ 慢速平台可以；高速机载 LiDAR 不够用
- 仓库：https://github.com/HKUST-Aerial-Robotics/A-LOAM

### 🟡 LIO-SAM

- **核心思路**：**因子图**（GTSAM 后端）+ 紧耦合 IMU + 帧到局部地图。
- **优势**
  - 紧耦合 IMU + LiDAR，**高速运动也能扛**
  - 因子图后端，回环因子可插拔
  - 配套 GPS 因子，机载有 GNSS 时直接融合
- **劣势 / 不 work 的场景**
  - 因子图求解**计算量较大**
  - **户外大场景必备**：室内、走廊点云退化场景会跪
  - GTSAM 内存占用不小
- **机载可行性**：✅ 户外 LiDAR 机载的**首选**
- 仓库：https://github.com/TixiaoShan/LIO-SAM

### 🟡 FAST-LIO / FAST-LIO2

- **核心思路**：**紧耦合 IEKF**（迭代误差状态卡尔曼滤波）+ iKD-Tree 数据结构。FAST-LIO2 取消了特征提取。
- **优势**
  - **速度极快**，HKUST 团队做了极致优化，**Orin 上 100+ FPS**
  - 紧耦合，**高速 / 高动态场景精度 SOTA**
  - FAST-LIO2 的 iKD-Tree 无需特征工程，对 LiDAR 通用性好
  - **真机验证充分**（FAST-LIO 论文里直接给的是无人机实验）
- **劣势 / 不 work 的场景**
  - 滤波法没有回环因子，**大场景长时间会漂**
  - iKD-Tree 内存随地图增长，长期运行要定期清理
- **机载可行性**：✅✅ **目前最推荐的 LiDAR 机载方案**
- 仓库：https://github.com/hku-mars/FAST_LIO

### 🟡 Fast-LVIO

- **核心思路**：**LiDAR + 视觉 + IMU 三模态紧耦合**，扩展自 FAST-LIO 框架。
- **优势**
  - LiDAR 退化场景（长直走廊、隧道）靠视觉救场
  - 视觉反哺 LiDAR 的几何标定
- **劣势 / 不 work 的场景**
  - 三模态**标定复杂**，外参漂移是大坑
  - 工程化不如 FAST-LIO 成熟
- **机载可行性**：✅ 多模态备选
- 仓库：https://github.com/hku-mars/Fast-LVIO

## 四、几个选型时的速查参考

| 场景 | 首选 | 备选 |
|:---|:---|:---|
| **室内小场景，CPU 即可** | ORB-SLAM3 / OV²SLAM | SVO |
| **户外无人机，视觉 + IMU** | VINS-Fusion | OpenVINS |
| **户外无人机，LiDAR + IMU** | FAST-LIO2 | LIO-SAM |
| **高速场景（> 10 m/s）** | FAST-LIO2 / SVO | LIO-SAM |
| **GNSS 可用 + 视觉** | VINS-Fusion + GPS 因子 | ORB-SLAM3 + GPS |
| **弱纹理 / 人造结构** | BASALT（线段） | ORB-SLAM3（特征调密） |
| **多模态 LiDAR+Visual** | Fast-LVIO | LVI-SAM |

## 五、本期小结

- **视觉惯性** 领域基本是 VINS-Fusion 和 OpenVINS 二选一，**前者工业项目多、后者工程化新**
- **LiDAR 惯性** 现在 FAST-LIO2 是公认的**机载首选**，HKUST 团队迭代很快
- **ORB-SLAM3** 仍然是最稳的视觉 SLAM 标杆，研究/产品都绕不开
- **SVO 系列** 在前向高速 FPV 场景不可替代
- 紧耦合 + 因子图后端是主流，纯滤波方法在精度上慢慢让位

下一期重点看：

- 有没有 FAST-LIO 后续工作（HKUST 团队）
- VINS 系列有没有重大更新
- 端到端学习 SLAM 是否有能在机载跑的工作
- 大模型/VLM 接入 SLAM 的具体方法（这部分主要看趋势，工程上还很远）
